Benchmarking Federated Synthetic Data Generators

An interactive dashboard to plan and track the project outlined by FHI.

Background & Motivation

Analyzing sensitive data in fields like healthcare and finance is critical, but privacy laws (e.g., GDPR) create significant barriers. Synthetic data, which mimics real data without exposing personal information, is a promising solution.

While centralized models (CTGAN, TVAE) exist, they require pooling data, which is often not feasible. Federated Learning (FL) offers a solution by training models collaboratively across institutions without data ever leaving the local site.

This project aims to systematically compare recently proposed federated models for generating synthetic tabular data.

Project Goal

To compare federated synthetic data generators (Fed-TGAN, VT-GAN, FedTabDiff) in a systematic way using open datasets.

Expected Outcomes

  • A working benchmarking pipeline built on the Flower framework.
  • A comparative evaluation across the four key metrics.
  • Practical guidelines on which model is best suited for different needs.
  • A final thesis/report documenting the methodology and results.

Core Tasks

1. Familiarization

Familiarize with federated learning (FL) and synthetic data generation techniques.

2. Benchmark Implementation

Implement a benchmark suite in the Flower framework for running federated experiments.

3. Train & Evaluate Models

Train and evaluate Fed-TGAN, VT-GAN, and FedTabDiff on selected public datasets (e.g., UCI Heart Disease, Breast Cancer Wisconsin).

4. Measure Performance

Measure performance across fidelity, privacy, fairness, and scalability.

5. Comparative Analysis

Compare federated models against each other. Centralized baselines (CTGAN, etc.) are included as reference points only.

6. Summarize & Recommend

Summarize findings in a clear and reproducible way, providing practical recommendations.

Evaluation Metrics

The comparison will be based on these four key pillars.

1. Fidelity

How realistic is the synthetic data? Does it capture the statistical properties, distributions, and correlations of the original data?

2. Privacy

How well is the sensitive information protected? This involves measuring the risk of membership inference attacks or other privacy breaches.

3. Fairness

Does the synthetic data exhibit the same biases as the real data, or does it amplify them? How does it perform across different subgroups (e.g., age, sex)?

4. Scalability

How efficient are the models? This includes training time, communication overhead between clients, and computational requirements.
